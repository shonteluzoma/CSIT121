but once you start to get in the groove of things it's really not that bad.
Having the basic understanding of how to structure HTML is important
in teaching you how webpages are built. Once you know your tags and
structures and work on your precision while typing to avoid errors, it should
be smooth sailing from there.

What I Learned...

  1. Attention to detail is key, always check for errors and typos.
  2. Use attributes like id, class, alt, and href properly for styling, accessibility, and navigation.
  3. Use helpful shortcuts the system provides like typing HTML:5 in the begining of your code to give you the correct layout you need.

Things that I learned since Module 02 for Module 03

  1. CSS lets you pick which HTML elements to style using selectors.
  2. When multiple rules apply to an element, specificity determines which one takes effect. ID selectors have the highest priority, next are class selectors, and then type selectors.
  3. Type selectors, ID selectors, and class selectors are the most popular. I used the last two in this webpage.
